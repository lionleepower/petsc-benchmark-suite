## Questions To Answer From The Figure

1. Runtime generally decreases as total cores increase, but not with ideal linear speedup.
2. At the same total cores, compare vertical position: lower runtime is better.
3. `T=1` is a strong high-core-count baseline; hybrid `T=2` or `T=4` can be competitive at intermediate core counts.
4. `T=8` often has more overhead and is less competitive in these runs.
5. The 1M case is more irregular, which is a useful sign that small problems can expose parallel overhead.
6. Larger problems show clearer scaling trends, but still remain far from ideal 128x speedup.

## How To Use This In The Report

Use the log-y figure when you want to compare trends across the whole runtime range.
Use the linear figure when you want to discuss absolute runtime differences.
