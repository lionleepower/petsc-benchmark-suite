### Technical Plan

* The first project milestone will focus on **building and testing PETSc with the OpenMP backend enabled**.
* Initial experimental workflow:

  1. Build PETSc using the OpenMP backend.
  2. Create or select a suitable matrix problem (initially simple and controlled).
  3. Run experiments under different configurations:

     * Vary number of nodes
     * Vary MPI ranks and OpenMP threads
  4. Measure execution time and compute **speedup** for comparison.
* Emphasis is on validating correctness first, followed by initial performance trends.

### Slurm / sbatch Configuration

* Discussed best practices for **MPI + OpenMP hybrid job submission**.
* Key points:

  * Explicitly set `--cpus-per-task` and ensure it is propagated to `srun`.
  * Configure OpenMP environment variables correctly:

    * `OMP_NUM_THREADS`
    * `OMP_PLACES=cores`
  * Use thread pinning options (e.g. `--hint=nomultithread`) to avoid oversubscription.
* A standard sbatch template will be used as a baseline for all experiments, with parameters adjusted systematically.
